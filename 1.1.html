รายการเรียนรู้ CSS

1: การเขียนแบบ External ช่วยให้โค้ดสะอา

คำอธิบาย: การแยก CSS ออกจากไฟล์ HTML (Separation of Concerns) ทำให้ไฟล์ HTML มีหน้าที่จัดการแค่โครงสร้าง (Structure) และเนื้อหาเท่านั้น

เหตุผล: เมื่อโค้ดไม่ปนกันจะช่วยลดความซับซ้อน (Complexity) ทำให้โปรแกรมเมอร์อ่าน Code ได้ง่ายขึ้น ลดโอกาสเกิดข้อผิดพลาดในการแก้ไขเนื้อหา และช่วยให้หาจุดที่ต้องแก้ไขได้เร็วขึ้น

2: สามารถใช้ไฟล์ CSS ร่วมกันได้หลายหน้า

คำอธิบาย: คุณสามารถเขียนไฟล์ style.css เพียงไฟล์เดียว แล้วนำไปเชื่อมต่อ (Link) กับหน้าเว็บกี่หน้าก็ได้ (เช่น หน้าแรก, หน้าเกี่ยวกับเรา, หน้าติดต่อ)

เหตุผล: สิ่งนี้เรียกว่า Reusability (การนำกลับมาใช้ใหม่) แทนที่คุณจะต้องเขียนโค้ดตั้งค่าฟอนต์หรือสีเดิมๆ ซ้ำๆ ทุกหน้า คุณแค่เขียนครั้งเดียวในไฟล์ External ทุกหน้าก็จะเปลี่ยนตามทันที

3: ง่ายต่อการแก้ไขดีไซน์ในอนาคต

คำอธิบาย: หากคุณต้องการเปลี่ยนสีธีมของทั้งเว็บไซต์จาก "สีน้ำเงิน" เป็น "สีเขียว" คุณแค่เข้าไปแก้ในไฟล์ CSS ภายนอกเพียงจุดเดียว

เหตุผล: สิ่งนี้เรียกว่า Maintainability (ความง่ายในการบำรุงรักษา) หากคุณใช้การเขียนแบบ Inline (เขียนใน Tag) และมีหน้าเว็บ 100 หน้า คุณต้องตามแก้ 100 จุด แต่การใช้ External CSS จะช่วยให้การปรับปรุงแบรนด์ (Rebranding) หรือการปรับหน้าตาเว็บทำได้ในเวลาไม่กี่วินาที
